コミュニティによる、コミュニティのための

io.js は Node.js と完全互換で npm のほぼ全てのモジュールと互換性があります。

io.js はコミュニティドリブンのオープンソースプロジェクトです。io.js は Node.js の主要なコントリビュータによって開始されましたが、既に多くの積極的な開発者を集めており、これは Node.js の歴史の中で最も多くなっています。

io.js に求められているものとは？

  1. 安定性
  2. 透明性 (デバッグとトレーシング)
  3. より良いストリーム
  4. 長期サポート
  5. 継続的に標準を受け入れる
  6. ローカライズ

関連するリストはロードマッププロジェクトにあります。コントリビュート歓迎です。

安定性ポリシー

私たちは他のどの言語よりも大きなパッケージエコシステムを持っており、我々はそれを壊すことはできません。

  • 私たちは JS API を削除しません
  • セマンティックバージョニング
    • API の追加はマイナーバージョンがインクリメントされることを意味します。
    • V8 の C++ API の変更は nan によってハンドリングされます。そのため、nan が変更を吸収できない場合は minor か major のバージョンが上がることになります。

安定性ポリシーのロードマップより抜粋。コントリビュート歓迎です！

チャンネル

Release 全てのバージョンがビルドされています。
安定しており、プロダクションで使えます。
Canary V8やその他の新機能が使える、毎晩ビルドされるバージョンです。
NG 次世代の ES6/7 の中心的な APIが使えます。


安定性ポリシーのロードマップより抜粋。コントリビュート歓迎です！

NG (ネクストジェネレーション)

  • モダンな JavaScript アプリケーションのための新しいプラットフォームを目指します。
  • 今後も下位互換性は残す予定です。
  • 新しい API は古いモジュールと API を壊すことなく ES6/7 を中心に据えることができます。
  • 現在のプラットフォームへの改善から今後の API のための作業への区切りとします。

NG の議論はNG repoで現在も進行中です。

デバッグとトレーシング

コアは domains のような高レベルの抽象化を目指すよりもむしろデバッグやトレースツールのような様々なエコシステムを利用可能にする必要があります。

  • trace_event との統合 (V8 と Chrome のための新しいトレース API )。
  • デバッグのエコシステムをドキュメント化し普及させます。
  • システムのトレーシングエンドポイントを統一します (DTrace, LTTngなど)

作業は Tracing Working Group に先導されています。コントリビュート歓迎です！

Stream

  • ストリームワーキンググループは readable-stream を引き継ぎ、これを io.js 公式の stream モジュールとしました。
  • 全ての既存の互換性の問題を修正しました。
  • ユーザエラーを回避するためにストリームの作成を簡素化します。
  • WHATWG Streamsを追求してインターフェースと互換性の問題を明確にする。
  • ストリームのパフォーマンスを向上させる。

作業はStreams Working Group に先導されています。コントリビュート歓迎です！

長期サポート

  • 新しいリリースラインは安定性と依存性のサポートと共にリリースされます。
  • 古いバージョンへのパッチリリースは人々が積極的にコントリビュートしているようにするために公開されます。
  • io.js は古いバージョンの依存関係について、V8 のように以前のリリースについてメンテナがサポートを停止する際に責任をとります。

Stability Policy の要約です。コントリビュート歓迎です！

ローカライズ

  • icu を含めた時の影響を軽減。(バイナリとメモリフットプリント)
  • 活気のあるローカライゼーションコミュニティを育成する。
    • 各言語のコミュニティは自律的です。(30以上がこれまでに登録されています)
    • メンバーシップを成長させる普及の取り組んでいます。(160人以上が取り組んでいます)
    • 翻訳ウェブサイト、API ドキュメント、インストーラやスライドの資料などの道具を向上させます。

あなたの言語のコミュニティにあなたも参加することができます :)
